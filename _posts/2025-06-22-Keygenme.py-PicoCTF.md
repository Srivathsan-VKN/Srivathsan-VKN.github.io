### Step 3: Identify the Key Validation Function
 
Found the `check_key()` function that validates license keys:
 
```python
def check_key(key, username_trial):
    global key_full_template_trial
    if len(key) != len(key_full_template_trial):
        return False
    else:
        # Check static base key part --v
        i = 0
        for c in key_part_static1_trial:
            if key[i] != c:
                return False
            i += 1
        # TODO : test performance on toolbox container
        # Check dynamic part --v
        if key[i] != hashlib.sha256(username_trial).hexdigest()[4]:
            return False
        else:
            i += 1
        if key[i] != hashlib.sha256(username_trial).hexdigest()[5]:
            return False
        else:
            i += 1
        if key[i] != hashlib.sha256(username_trial).hexdigest()[3]:
            return False
        else:
            i += 1
        if key[i] != hashlib.sha256(username_trial).hexdigest()[6]:
            return False
        else:
            i += 1
        if key[i] != hashlib.sha256(username_trial).hexdigest()[2]:
            return False
        else:
            i += 1
        if key[i] != hashlib.sha256(username_trial).hexdigest()[7]:
            return False
        else:
            i += 1
        if key[i] != hashlib.sha256(username_trial).hexdigest()[1]:
            return False
        else:
            i += 1
        if key[i] != hashlib.sha256(username_trial).hexdigest()[8]:
            return False
 
        return True
```
 
### Step 4: Understand the Key Validation Logic
 
The function validates the key in two parts:
 
**Static Part:** The key starts with a fixed prefix defined in `key_part_static1_trial`, which matches `"picoCTF{1n_7h3_kk3y_of_"`
 
**Dynamic Part:** After the static part, 8 characters are validated against a SHA256 hash of the username. The validation checks these specific hash indices in this order:
- `hashlib.sha256(username_trial).hexdigest()[4]`
- `hashlib.sha256(username_trial).hexdigest()[5]`
- `hashlib.sha256(username_trial).hexdigest()[3]`
- `hashlib.sha256(username_trial).hexdigest()[6]`
- `hashlib.sha256(username_trial).hexdigest()[2]`
- `hashlib.sha256(username_trial).hexdigest()[7]`
- `hashlib.sha256(username_trial).hexdigest()[1]`
- `hashlib.sha256(username_trial).hexdigest()[8]`
**Explanation:** The code converts the username into a SHA256 hash, then takes characters from positions 4, 5, 3, 6, 2, 7, 1, and 8 from the hexdigest string (in that order) to create the dynamic portion of the key.
 
### Step 5: Extract the Hash Values
![](/assets/blog/Keygenme.py/Cyberchef.png)
Using `bUsername_trial` (the variable used in `enter_license()`), calculated the SHA256 hash. The hexdigest output is `ba6c084a4...`
 
Mapped the indices to extract the required characters:
- Index [4] → '0'
- Index [5] → '8'
- Index [3] → 'c'
- Index [6] → '4'
- Index [2] → '6'
- Index [7] → 'a'
- Index [1] → 'a'
- Index [8] → '4'
This gives the dynamic part: `08c46aa4`

## Key Insight
 
The license key validation combines a hardcoded static prefix with a dynamic suffix derived from the username's SHA256 hash. By understanding this pattern and extracting characters from the hash at the correct indices, the valid key can be reverse-engineered without needing to guess or brute force.

## Conclusion
 
By analyzing the `check_key()` function and understanding how it validates both static and dynamic key components, the flag was successfully extracted. The key insight was recognizing that the validation uses specific character positions from a SHA256 hash of the username to create the dynamic portion of the license key.
